1. Имеется опыт ручного тестирования в соответствии с резюме.
2. Опыта нагрузочного тестирования нет.
3. Опыта тестирования безопасности нет.
4. Для решения тестового задания использовал Postman и справку по newman.
5. Команда для запуска тестов ```newman run https://raw.githubusercontent.com/roman-albertovich/testing_dummyjson/refs/heads/main/Test.postman_collection.json```
#### Список реализованных проверок:
1. get_user_info_without_token:
  - проверка, что сервис отвечает 401 на запрос без токена.
  - проверка, что сервис возвращает понятный текст с ошибкой для пользователя.
2.  auth:
-  проверка, что сервис отвечает 200 на корректные реквизиты
-  проверка, что сервис возвращает accessToken и этот токен не пустой
3. Auth 400 bad password:
 - проверка, что сервис отвечает 400 на некорректные резвитиы пользователя
 - проверка, что сервис возвращает понятный текст с ошибкой для пользователя
 - проверка, что accessToken отсутствует в ответе
4. get_user_info:
 -  проверка, что сервер возвращает профиль авторизованного пользователя
5. carts_user_200:
  - проверка, что идентификатор пользователя поля в корзине совпадает с идентификатором указанного в запросе.
6. carts_id_200:
  - проверка, что сервис отвечает 200 на корректные реквизиты. Можно добавить проверку на соответствие id корзины в запросе и ответе.
7. carts_add_201:
  - проверка, что при создании данных сервер возвращает код 201
  - проверка, что в ответе id корзины присутствует и не пустой
  - проверка, что в ответе userId пользователя присутствует и не пустой
  - проверка, что в ответе userId пользователя совпадает с userId в запросе.
8. get_card_by_id:
  - проверки, что в ответе Id корзины совпадает с Id в запросе.
  - проверки, что поля userId, discountedTotal, totalProducts и totalQuantity совпадают с указанными в запросе на добавление корзины "carts_add_201". Тесты падают, так как API доступен только для чтнения и не сохраняет данные на изменение/добавление.
9. get_card_with_uncorrected_id
  - проверка, что сервис отвечает 400 на некорректные резвитиы корзины
  - проверка, что сервис возвращает понятный текст с ошибкой для пользователя
10. get_card_by_user:
  - проверки, что поля userId, discountedTotal, totalProducts и totalQuantity совпадают с указанными в запросе на добавление корзины "carts_add_201". Тесты падают, так как API доступен только для чтнения и не сохраняет данные на изменение/добавление.
11. patch_carts:
  - проверка, что данные в ответе соответствует переданным значениям.
12. delete_cart:
  -  проверка, что корзина помечена как удалённая
